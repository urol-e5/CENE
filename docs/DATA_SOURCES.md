# Data sources

Every dataset used by the Coral Epigenetic Network Explorer maps to a real output
file in the analysis repository
[`urol-e5/deep-dive-expression`](https://github.com/urol-e5/deep-dive-expression)
(branch `main`) or to a value transcribed from the manuscript. No results, sample
sizes, interactions, annotations, or statistics were invented.

Species codes: `Apul` = *Acropora pulchra*, `Peve` = *Porites evermanni*,
`Ptuh` = *Pocillopora tuahiniensis*.

## Source → site component map

| Site component | Generated file(s) | Source in `deep-dive-expression` | Transformation |
| --- | --- | --- | --- |
| **Network Explorer** (nodes, edges, shapes, filters) | `network/{sp}.json` | `M-multi-species/output/15-miRNA-mRNA-lncRNA-network-ceRNA/{sp}_nodes_..._p0.05.csv` and `{sp}_edges_..._p0.05.csv` | Parse; join node type/status; classify each edge (interaction class, direction, evidence, epi-miRNA/ceRNA flags, epi category); compute degrees. |
| **Evidence Table** | (reads `network/{sp}.json` edges) | same as above | Denormalized edge records reused directly. |
| **Epigenetic Machinery Explorer** | `epimachinery.json` | `M-multi-species/output/12-miRNA-epimachinery/miRNAtargets_mach.csv` | Parse; normalize one category label (`and` → `&`); derive protein short name and direction; category summaries. |
| — protein-name join for epi nodes | (into `network/{sp}.json`) | `M-multi-species/output/09.1-epimachinery-ncRNA-protein-expression/ncRNAepimachinery_gene_db_spec.csv` | Reference (not currently required — labels come from `miRNAtargets_mach.csv`). |
| **Compare Species** cards & charts | `summary.json` | Manuscript Results + computed network stats | Manuscript headline values (constants in `scripts/build_web_data/lib.mjs`) + counts computed from the loaded p<0.05 network. |
| **miR-100 panel** | `mir100.json` | network edges + `miRNAtargets_mach.csv` + manuscript | Compute per-species interaction / direction / region counts for miR-100; epi targets + enrichment from manuscript. |
| **Methylation Landscape** | `methylation.json` | `{D-Apul/output/08-Apul-WGBS, E-Peve/output/12-Peve-WGBS, F-Ptuh/output/12-Ptuh-WGBS}/bismark_cutadapt/global_methylation_levels.txt` + manuscript | Per-sample global CpG values (×100 to %); feature-level levels transcribed from manuscript text (numeric per-feature table unavailable — see gaps). |
| **Regulatory Story** | `regulatory-stories.json` | Manuscript (Fig. 4, Table 2) | Curated chains encoded from the manuscript. |
| **Downloads** | `downloads.json` | mixed | Catalogue with sizes + source links. |
| **Interactions (raw download)** | (linked to source) | `M-multi-species/output/20-supplementary-files/miRanda_PCC_{mRNA,lncRNA}_sig.csv` | Vendored, offered for download. |
| Manifest / dictionaries / build summary | `data-manifest.json`, `dictionaries/network.json`, `build-summary.json` | — | Generated by the pipeline. |

## Known data gaps

1. **Per-feature methylation numbers.** `10-Cross-Species-Methylation/` contains
   only figures (JPEG); no machine-readable per-feature mean-methylation table.
   The Methylation view therefore shows global values (numeric) plus qualitative
   feature levels transcribed from the manuscript, clearly labelled *awaiting data*.
2. **General gene annotation.** The network node tables carry no functional
   annotation for non-epigenetic-machinery genes; those appear by transcript ID.
   The topGO enrichment outputs in `04.2-miRNA-comparison-targets-FE/` could be
   joined to add this later.
3. **miRanda score on network edges.** The integrated network edge files carry
   binding *energy* and alignment but not the miRanda score column (that lives in
   the `20-supplementary-files` miRanda tables under ShortStack `Cluster_*` IDs,
   which use a different identifier system than the named network). Energy is shown
   as the binding-strength metric.
